# 数学知识星图扩充总路线

## 当前范围

按最新收缩方案执行：

- 初中数学：保留较完整的核心知识网络
- 高中数学：只保留高一中最重要、最适合与初中衔接的函数、三角、向量与几何主线
- 目标优先级：先保证整体稳定运行，再逐步增加内容

当前目标规模调整为：

- 80 到 140 个高价值节点
- 180 到 320 条高解释性关系
- 每个核心概念至少 3 条有效关系
- 尽量减少“堆数量但前端没接住”的无效扩充

## 不再确认规则

用户已授权后续阶段直接执行，不再重复请求确认。

每个阶段都必须：
1. 先读取本文件、`PROJECT_HANDOFF.md`、`data/README.md`、`data/schema.json`、`data/catalog.json` 和已有 seeds。
2. 只在统一 schema 下扩充或整理知识数据。
3. 完成后运行 `npm.cmd run validate:data`。
4. 完成后运行 `npm.cmd run build`。
5. 更新本路线文件和交接文件中的状态、统计和验证结果。

## 数据质量底线

- ID 稳定、可读、全局唯一。
- 同义词写入 `aliases`，不能重复建节点。
- 关系两端必须存在。
- 每个 `core_concept` 至少有 3 条有效关系。
- 所有关系都必须填写 `reason`。
- 所有新增内容必须通过校验工具。

- 自动续作必须先读 `PROJECT_HANDOFF.md`。

- 若无法读取精确 token 数，则在保守估算接近 180000 token 时提前迁移。
